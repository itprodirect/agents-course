## Quickstart

### 1) Clone the repo

```bash
git clone https://github.com/itprodirect/agents-course.git
cd agents-course
```

### 2) Create & activate a virtual environment

#### Option A: `uv` (recommended)

```bash
uv venv .venv
uv sync
```

#### Option B: `pip`

```bash
python -m venv .venv
# Windows (Git Bash)
source .venv/Scripts/activate
# macOS/Linux
# source .venv/bin/activate

pip install -r requirements.txt
```

### 3) Set environment variables

Create a `.env` file in the repo root:

```env
# Only set if you’re on Dedicated Cloud / on-prem
WANDB_BASE_URL=

OPENAI_API_KEY=your_openai_api_key
WANDB_API_KEY=your_wandb_api_key
```

Load it:

**macOS/Linux**

```bash
set -a
source .env
set +a
```

**Windows (Git Bash)**

```bash
set -a
source .env
set +a
```

(Or set variables however you prefer; the scripts just need the keys present in the environment.)

## Running the MCP submission script

The submission script expects these folders:

- `sample_files/` (input files)
- `outputs/` (generated outputs)

The repo already contains `sample_files/`. When you run the script, it will create `outputs/` if needed.

Run:

```bash
python _6.1_mcp.py
```

Then open the Weave trace in W&B:

- In W&B: **Project → Weave → Traces**, click the latest call
- Use the shareable link (public project), and paste it into the submission template file

## Troubleshooting

### ZoneInfoNotFoundError: “No time zone found with key UTC”

If you encounter this while running eval scripts:

1. Ensure `tzdata` is installed:

```bash
pip install tzdata
```

2. Windows users may need:

```bash
pip install tzdata zoneinfo-backport
```

3. For Python < 3.9:

```bash
pip install backports.zoneinfo tzdata
```

4. Workaround: set TZ before running:

**macOS/Linux**

```bash
export TZ=UTC
python _5_evals.py
```

**Windows (Command Prompt)**

```bat
set TZ=UTC
python _5_evals.py
```

**Windows (PowerShell)**

```powershell
$env:TZ="UTC"
python _5_evals.py
```
